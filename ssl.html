Mark SSL Certificates as Untrusted in Chrome

Download SSL Certificates

This tutorial explains how to import specific SSL certificate files from a ZIP archive into Chrome's Certificate Manager so that they become untrusted. This process is useful for educational or testing purposes.

Step 1: Extract the ZIP Archive

Start by unzipping your archive. The extracted folder should contain multiple certificate files—for example:

  • Starfield Root Certificate Authority - G2.pem
  • Amazon Root CA 1.pem
  • Amazon Root CA 2.pem
  • Amazon Root CA 3.pem
  • Amazon Root CA 4.pem

Step 2: Open Chrome’s Certificate Manager

Open Google Chrome and enter the following URL in the address bar:

chrome://certificate-manager/

This opens Chrome's built-in certificate management window.

Step 3: Locate Your Local Certificates

Once inside the Certificate Manager, click on the Local Certificates tab. Within this section, look for the category labeled Installed by You where local certificates are managed.

Step 4: Import the Distrusted Certificates

Find the Import button next to the distrusted SSL section. Click it, then select the following certificate files from your extracted folder:

  • Starfield Root Certificate Authority - G2.pem
  • Amazon Root CA 1.pem
  • Amazon Root CA 2.pem
  • Amazon Root CA 3.pem
  • Amazon Root CA 4.pem

After importing, Chrome will mark these certificates as untrusted, preventing secure connections that use them.

Step 5: Test and Confirm

Visit a website that depends on one of these certificates. You should receive a warning from Chrome about an untrusted connection, confirming that the certificates are no longer trusted.

Warning: Changing trusted certificate settings can affect your secure browsing. Always make sure you understand the implications and keep backups of your original certificates.
